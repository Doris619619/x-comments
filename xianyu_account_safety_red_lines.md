# 闲鱼账号安全红线

以下规则必须严格遵守，任何一条都不得实现。

## 禁止行为

- 禁止高频抓取、无限翻页、无限滚动。
- 禁止多个任务同时使用同一个闲鱼账号。
- 禁止同一关键词重复并发采集。
- 禁止失败后连续快速重试。
- 禁止每 5 分钟全量检查所有商品。
- 禁止自动识别或绕过验证码。
- 禁止绕过登录、风控、访问限制或安全验证。
- 禁止破解接口签名、伪造请求参数或模拟内部接口。
- 禁止使用代理池、IP 轮换、多账号轮换继续抓取。
- 禁止在出现 403、429、访问频繁提示后继续请求。
- 禁止抓取私聊、手机号、精确地址等非公开个人信息。
- 禁止把账号密码、短信验证码、Cookie、Token 写入代码、日志或 Git 仓库。
- 禁止把 Playwright 登录态文件上传到 GitHub 或发送给他人。

## 必须立即停止的情况

出现以下任意情况，立即停止当前任务，禁止自动重试：

- 出现验证码；
- 出现安全验证；
- 登录状态失效；
- 页面跳转到登录页或验证页；
- HTTP 403 或 429；
- 页面提示访问频繁、账号异常或操作受限；
- 搜索结果异常归零；
- 连续请求失败；
- 页面结构异常，无法确认抓取结果是否正常。

停止后将任务标记为：

```text
blocked_by_auth_or_risk_control
```

等待人工检查后才能继续。

## 账号凭据保护

账号必须由开发者在本地浏览器中手动登录。

不得要求开发者向 Codex 提供：

- 账号密码；
- 短信验证码；
- Cookie；
- Token；
- 完整登录态文件。

登录态文件必须加入 `.gitignore`：

```gitignore
.env
state/
playwright/.auth/
*.storage_state.json
```

账号安全优先于采集成功率。遇到风险提示时，宁可停止任务，也不得继续尝试。
