# 已知限制与安全规则

## 当前限制

- Goal 2 已使用有头 Playwright 完成两次 50 条真实采集和随机十条抽查；无头模式仍会出现“非法访问”，因此当前真实任务必须使用有头正常浏览器。
- 不保证抓取全部商品，只允许后续 Goal 2 的 3 页或最多 50 条有限范围。
- 尚未实现每 5 分钟调度、商品售出/下架复查或搜索消失判断。
- SQLite 适合本地 POC；PostgreSQL 只完成驱动、URL 和迁移兼容预留。
- 进程内任务队列不提供分布式可靠投递。
- 登录态会过期，需要用户在本机重新人工登录。
- 闲鱼页面字段和结构可能变化，离线 fixture 通过不能替代真实抽查。
- 未登录页面可能返回与关键词不匹配的公共推荐，不能作为任务成功证据。

## 强制停止条件

出现验证码、安全验证、登录失效、登录/验证页跳转、HTTP 403/429、访问频繁、账号异常、操作受限、搜索结果异常归零、连续请求失败或关键结构异常时，任务必须停止并标记 `blocked_by_auth_or_risk_control`，禁止自动重试。

## 明确不实现

验证码识别/绕过、签名破解、访问控制绕过、代理池、多账号轮换、高频并发、私聊采集、手机号采集和精确地址采集均不在项目范围内。

`storage_state.json` 是敏感本地文件，已被 Git 忽略；不得提交、上传、打印或写入数据库。
