## 9. 风险列表

1. 未登录页面可能返回与关键词无关的公共推荐；
2. 登录状态可能过期，且真实采集可能触发验证码或风控；
3. 搜索响应字段和页面选择器可能变化；
4. 搜索结果 API 是页面内部实现细节，必须由正常页面访问触发，不应签名破解或直接伪造请求；
5. 两个参考仓库无明确许可证，任何代码复制都有版权风险；
6. 参考仓库中的多账号/代理轮换与本项目安全红线冲突；
7. 进程内队列在应用重启时不具备可靠投递能力，这是 POC 有意限制；
8. 本阶段不能证明 Goal 2 的实际数量和字段正确率，必须届时真实运行并人工抽查。

## 10. 未解决问题

- 登录后搜索响应在用户环境中的具体稳定性，需要 Goal 2 实测；
- 当前页面最终采用 3 页还是 50 条边界，需要 Goal 2 根据实际分页机制记录；
- 闲鱼平台条款和允许的自动化范围需由项目责任人持续评估；技术实现始终遵循最小访问、有限范围和立即停机红线。

这些问题不阻塞 Goal 1 的纯离线骨架，但在 Goal 2 通过前不能声称 POC 已完成。
